# Dream Interpretation Rules — 해몽 규칙 레퍼런스

> Dreamy 서비스의 꿈 해석 품질·데이터 구조·문화 가드레일을 한 곳에 모은 문서.
> `lib/claude.ts` 프롬프트 구성, few-shot 예시, 톤 튜닝, UI 렌더 스펙에 재사용된다.

---

## 1. 해석 프레임 (4단 구조)

| 단계 | 역할 | 예 |
|------|------|-----|
| 1.1 풍경 묘사 | 사용자의 꿈 장면을 부드럽게 재서술하여 공감대 형성 | "복숭아 나무 아래 구렁이가 조용히 바라보는 장면이네요." |
| 1.2 상징 해석 | 문화권 전통/민속 기반 상징 의미 | "복숭아는 생명·복, 구렁이는 용의 기운을 품은 보호자예요." |
| 1.3 심리 의미 | 현재의 감정·무의식 반영 (Freud·Jung 관점) | "새로운 시작에 대한 기대와 긴장이 공존해요." |
| 1.4 현실 조언 | 오늘·이번주에 할 수 있는 작은 행동 | "가까운 사람과 소식을 나눠보세요." |

- **basic 해석**: 4단을 3~4문장이 아니라 **800~1,600자** 정도로 풍부하게. (§10 참고)
- **premium 그림일기**: 5페이지로 분할되는 건 *해석이 아니라 꿈 본문*. 해석은 별도 블록 구조. (§4·§5 참고)

## 2. 문화권별 해몽 전통

### 2.1 한국 — 민속 + 주공해몽 수용
- **핵심 원리**: 해음(동음이의) + 취상비류(象類)
- **태몽**: 임신 전후의 꿈이 태아의 성별·미래를 암시 (동양권 고유 개념)
- **대표 길몽**: 용, 돼지, 똥, 복숭아, 구렁이, 호랑이, 거북이, 잉어, 고추
- **대표 흉몽**: 이빨 빠짐, 신발 잃어버림, 머리카락 잘림, 도망, 추락
- **어조**: 존댓말, 경고성은 완곡하게

### 2.2 중국·대만 — 周公解梦
- 한국 해몽의 원류. 민간 바이블
- **해음 해석**: 棺=官(관직), 材=財(재물)
- **뱀**: 재물·지혜 / **물고기**: 재운
- 대만은 전통 주공해몽 + 서양 심리학 혼합

### 2.3 일본 — 夢占い
- **하츠유메(初夢)**: 새해 첫 꿈으로 한 해 운세 점침
  - 최고 조합 **一富士二鷹三茄子** (후지·매·가지 = 無事·高·成す)
- **자연·계절 비중** 큼 (벚꽃=전환기)
- **물**: 맑음=감정 명료 / 탁함=미해결 감정

### 2.4 서구 — Freud · Jung · Modern
- **Freud**: 꿈은 억압된 욕망의 위장된 표현
- **Jung**: 꿈은 무의식의 솔직한 메시지 / 집단 무의식의 원형(archetype)
- **Jung 대표 원형**: falling, flying, being chased, losing teeth, naked in public, exam unprepared
- **현대 인지**: 기억 통합·감정 처리

## 3. 상징 사전 (문화권 교차 매핑)

## 4. 세부 분기 테이블

### 4.1 이빨 꿈 (한국 민속 기준)
| 위치 | 피 없음 | 피 남 |
|------|---------|-------|
| 윗니 | 부모·어른의 건강/관계 주의 | 큰 사고·중병 가능성 |
| 아랫니 | 형제·친구 소소한 변화 | 형제·친구의 사고·질병 |
| 앞니(1개) | 가까운 어른 | 큰 변고 |
| 앞니(2개) | 가족 전반 | 큰 사건 |
| 어금니 | 지인 재물·건강 | 심각한 손실 |
| 전부 | 관계·일 재편 | 중대한 재난 (톤 주의) |

- **Jung 관점 병기 권장**: "변화와 성장의 전환기"로 긍정 해석 여지 확보.

### 4.2 태몽
- **받음**: 품에 안김 → 수태 암시
- **먹음**: 완전한 수용 → 아이 기운 흡수
- **쫓김**: 격렬 → 장래 큰 인물
- **놓침**: 유산·기회 상실 (톤 주의)
- **성별 민속**: 복숭아·딸기·꽃 → 딸 / 고추·밤·호박 → 아들 / 구렁이 → 남아 / 용 → 뛰어난 아들(전통) 혹은 대업(현대)

### 4.3 추락
높이 높음 / 착지함 / 끝없이 / 물·침대로 떨어짐 — 변수별 의미 차이

### 4.4 쫓김
추격자 유형(얼굴 없는 사람 · 아는 사람 · 맹수 · 괴물/그림자) × 잡힘 여부

## 4.5 길몽 / 흉몽 / 중립몽 분류 (Auspice)

모든 꿈은 **auspice** 로 분류되어 카드 디자인·뱃지 컬러가 바뀐다.

### 분류 기준 (AI가 판단)
| 분류 | 대표 신호 | 카드 디자인 |
|------|-----------|-------------|
| **auspicious (길몽)** | 용·돼지·복숭아·잉어·맑은 물·태몽·선물 받음·비행 | 따뜻한 골드·핑크 그라데이션, 소프트 글로우 |
| **ominous (흉몽)** | 이빨 빠짐·추락·쫓김·피·신발 분실·귀신·다리 못 움직임 | 어두운 와인 톤, 무거운 그림자 |
| **neutral (중립몽)** | 평범한 일상, 모호한 장면, 감정 약함 | 기본 네이비 글래스 카드 |

### UI 분리 원칙
- **카드 외부(헤더)**: auspice 뱃지 = "길몽" / "흉몽" / "중립몽" + 타입 뱃지 "그림일기" / "기본"
- **카드 내부(콘텐츠)**: 사용자가 선택한 Mood pill (행복한/이상한/불안한 등)
- 이유: auspice = AI의 객관적 분류 / mood = 사용자의 주관적 감정 → 분리해서 혼동 방지

### 폴백
AI 응답에 auspice 가 없거나 null 이면 `inferAuspiceFromMood(mood)` 로 추정:
- happy/excited/peaceful/fascinating → auspicious
- scary/anxious/sad/confused → ominous
- weird/nostalgic/null → neutral

## 5. Premium 데이터 구조 스펙 (UI 렌더와 1:1)

Dreamy 의 **premium(그림일기)** 응답은 단순한 "긴 해석"이 아니라 아래 세 부분이 각각 별도 필드로 나뉜다.

```ts
interface DreamEntry {
  dream: string              // 사용자 원문
  pages: DiaryPage[]         // 꿈을 5장으로 쪼갠 그림일기 (꿈 '본문' 재구성)
  interpretationBlocks: InterpretationBlock[]  // 상세 해석 블록 6~8개
  lucky: LuckyToday          // 오늘의 행운 아이템 + 색상
  ...
}

interface DiaryPage {
  title: string              // 5~8자 함축적 제목
  illustration: string       // 단일 이모지 (일러스트)
  text: string               // 2~3문장, 반말 친근체
}

interface InterpretationBlock {
  emoji?: string             // 좌측 일러스트 이모지
  heading?: string           // 소제목 (선택)
  body: string               // 본문 (마크다운 **볼드** 지원), 존댓말
}

interface LuckyToday {
  item: string               // 가지고 다니면 좋은 구체 아이템
  colorName: string          // 한글 색상 이름
  colorHex: string           // #RRGGBB
  advice: string             // 1문장, 왜 오늘 이 조합인지
  avoid: string[]            // 오늘 피해야 할 것 3~4개 (구체적 행동 형태)
  luckyDirection?: string    // 행운의 방향 (동/서/남/북/남동 등)
  luckyNumber?: number       // 행운의 숫자
}
```

**중요: `pages` 는 해석이 아니다.** 사용자의 꿈 본문을 감각·감정을 유지한 채 5장의 일러스트 일기로 쪼갠 것. 해석은 `interpretationBlocks` + `lucky` 가 담당한다.

## 6. 그림일기 분할 원칙 (pages)

### 분할 기준
- **5장 고정**. 많지도 적지도 않게.
- 시간 순서 유지 (장면 1 → 장면 2 → ...)
- 각 페이지는 **하나의 장면/모티프**만 담기 (여러 사건을 섞지 않기)
- 원문에 없는 요소는 추가 금지. **원문의 감각·감정을 확장하는 방향**으로만.

### 페이지 작성 규칙
- **제목**: 5~8자 함축 ("깜빡이는 복도", "뒤따르는 발소리")
- **일러스트**: 단일 이모지 1개. 해당 장면의 상징을 담음.
- **본문**: 2~3문장 / 반말 친근체 ("~이었어", "~이야")
- 페이지 번호는 UI 에서 `n / 5` 로 자동 표시

### 어조
사용자의 꿈일기 페이지이므로 **반말 친근체**. `interpretationBlocks` 의 존댓말과 명확히 구분한다.

### 예시 (seed-2 복도 꿈)
| # | 제목 | 일러스트 | 본문 요약 |
|---|------|---------|-----------|
| 1 | 깜빡이는 복도 | 🕯️ | 형광등 깜빡이는 복도 묘사 |
| 2 | 뒤따르는 발소리 | 👣 | 발소리가 걸음에 맞춰 따라옴 |
| 3 | 늘어나는 문 | 🚪 | 돌아볼수록 문이 늘어남 |
| 4 | 창문 속 다른 나 | 🪞 | 창문 속 얼굴이 자기가 아님 |
| 5 | 불린 이름 | 🔔 | 뒤에서 이름이 불리고 다리가 안 움직임 |

## 7. 해석 블록 규칙 (interpretationBlocks)

### 구성
- **블록 개수**: 6~8개 권장 (너무 적으면 피상적, 너무 많으면 산만)
- 각 블록 = `emoji + heading + body`
- body 는 **볼드(`**...**`)** 로 핵심 키워드 강조

### 권장 블록 순서 (구구절절함 유지)
1. **꿈의 첫인상** — 가장 특이한 모티프 짚기. "이 꿈이 왜 특별한지" (§13 첫인상 규칙)
2. **핵심 상징 1** — 가장 강한 상징 풀이
3. **핵심 상징 2** — 두 번째 상징
4. **(선택) 핵심 상징 3**
5. **심리학적 의미** — Jung·Freud·Modern 중 적합한 프레임 선택
6. **감정의 신호** — 꿈 속 감정 vs 깬 후 감정의 차이 해석
7. **오늘 당신에게** — 실행 가능한 조언 (1~3개)

### 톤
- **존댓말** ("~예요", "~이에요")
- 한국 민속 + 심리학 교차 언급 권장
- **"~ 일 수 있어요"** "~ 로 풀이돼요" 같은 가능성 표현 선호

## 8. 오늘의 행운 규칙 (lucky)

### 필드별 가이드
- **item**: "작은 녹색 화분 또는 라벤더 향 사쉐" 처럼 **구체적이고 구하기 쉬운** 아이템. 추상(예: "용기") 금지.
- **itemEmoji**: 단일 이모지, 아이템 시각화
- **colorName**: 한국어 색상 이름 ("세이지 그린", "따뜻한 베이지")
- **colorHex**: `#RRGGBB` — UI 에서 컬러칩과 그라데이션에 사용
- **advice**: 1문장, 왜 이 조합이 오늘 필요한지

### 선택 원리: "보완 관계"
꿈의 기분과 반대/보완되는 톤을 고른다.

| 꿈 무드 | 추천 방향 | 색상 예 |
|---------|-----------|---------|
| scary / anxious | 그라운딩·안정 | 세이지 그린, 웜 베이지, 우드 톤 |
| sad / nostalgic | 부드러운 온기 | 피치 오렌지, 소프트 옐로우 |
| fascinating / excited | 차분한 깊이 | 네이비, 인디고, 에메랄드 |
| peaceful / happy | 선명한 포인트 | 코랄 핑크, 머스터드 |
| weird / confused | 명료함·정돈 | 클리어 화이트, 스틸 그레이 |

### 피해야 할 것 (avoid) 작성 규칙
- 3~4개, 구체적 행동 형태 ("~하는 것", "~을 삼가세요")
- 예: "중요한 결정 몰아서 하기", "새벽 1~3시 SNS 스크롤", "카페인 과다 섭취"
- 꿈의 모티프에서 파생되는 실질 조언 우선 (거울 꿈 → "거울 오래 응시하지 않기")
- UI 상 별도 카드로 분리되어 표시됨 (본문 안에 녹이지 말 것)

### 방향·숫자
전통 해몽 + 현대 운세 블렌드. 방향은 8방위 중 하나, 숫자는 1~9 중 하나 권장.

### 금지
- 운세 수비학·수점술 깊게 파지 않기
- 고가 아이템 권유 금지 ("100만 원짜리 ~")
- 특정 브랜드 명시 금지

## 10. 해석 구구절절함 원칙 (LLM 출력 품질)

피상적 해석 vs 구구절절한 해석의 차이 = **구체성 × 다층성 × 연결성**.

### 구조 권장 (Basic 해석, 200~400자가 아닌 800~1,600자)
1. **오프닝 (1~2문장)** — 꿈의 특이점 거울처럼 짚기
2. **상징 해석 (불릿)** — 각 상징 개별로, **볼드**로 키워드 강조
3. **심리적 의미 (2~3문장)** — Jung/Freud/현대 중 적합한 프레임
4. **현재 상태 추정 (1~2문장)** — 조심스러운 제안
5. **오늘의 조언 (불릿 2~3개)** — 실행 가능한 작은 행동
6. **마무리 (1문장 + 이모지 1~2개)**

### 연결성
앞 단계가 다음의 근거가 되는 인과 흐름. 사전식 나열 ❌.

### 다층성
한국 민속 + 심리학 + 현재 상황 **3개 층**을 섞을 때 가장 풍부. 충돌 해석은 **한국 기본 + 서구 Jung 병기**.

### 금지
- 반복 표현 / 일반론 남발 ("꿈은 무의식의 반영") / 공포 조장 / 의료·재무·법률 구체 자문

### Canonical Basic 포맷 (모든 basic 해석에 동일 적용)
```
[오프닝 — 1~2문장으로 꿈의 보편성·핵심 모티프 짚기]

상징 해석
• [구체적 상징 1]은 "…" 감각/뜻/표현이에요.
• [구체적 상징 2]는 …
• [구체적 상징 3]는 … (필요 시)

심리적 의미
[2~3문장. 내면 기준·압박·감정 처리 관점. Jung 원형 언급 가능]

오늘의 조언
[1~2문장. 실행 가능한 작은 제안. 따뜻한 톤]
```

- **소제목 "상징 해석" / "심리적 의미" / "오늘의 조언" 은 평문으로 작성** (마크다운 ** 없이).
  프론트엔드 `FormattedInterpretation` 렌더러가 이 이름을 자동 감지해 **볼드 헤더로 스타일링**함 (크기 14, fontWeight 700, 밝은 색).
- 본문 안 핵심 키워드에만 인라인 **볼드** 사용.
- 이모지 금지.
- 전체 500~900자.
- 예시: §18.1 "시험 꿈" 참고.

### 렌더러 자동 헤더 감지
`components/dream/DreamDetailModal.tsx` 의 `CANONICAL_HEADINGS` 배열에 포함된 소제목은 단락 첫 줄일 때 자동으로 볼드로 렌더. 프롬프트·시드 데이터에 `**...**` 를 둘러쌀 필요 없음.

현재 감지 목록:
- `상징 해석`, `심리적 의미`, `오늘의 조언` (canonical)
- `상징 풀이`, `심리학적 의미`, `현재 당신의 상태`, `한국 전통 해몽` (레거시 호환)

## 14.4 Mood 멀티 셀렉트 + 자동 태깅

### 입력
- MoodSelector 는 **복수 선택**. 사용자가 여러 기분을 동시에 고를 수 있음.
- 타입: `DreamEntry.moods: Mood[]` (단일 `mood` 필드는 deprecated)
- 0개 선택도 허용 — AI 가 꿈 내용을 분석해 자동 태깅

### 자동 태깅 규칙
- `moods` 가 빈 배열이면 AI 응답에서 돌아온 `moods` (1~3개)를 그대로 사용
- `moods` 를 사용자가 고르면 그대로 저장 (AI 는 참고만)
- 자동 태깅된 mood 도 UI 상 똑같이 MoodPill 로 노출

### 렌더
- 카드/모달 내 MoodPill 은 **배열 반복 렌더**로 여러 개 동시 표시
- `flex-wrap` 으로 줄바꿈
- MoodSelector props: `values: Mood[]`, `onChange: (Mood[]) => void` — 토글 방식

### auspice 추정
`inferAuspiceFromMoods(moods[])`:
- 긍정 무드 개수(pos) > 부정(neg) → auspicious
- neg > pos → ominous
- 같거나 모두 neutral → neutral

## 14.44 그림일기 AI 이미지 생성 (Pollinations → Picsum 폴백)

### 파이프라인 (Next.js 서버 프록시 경유)
1. 클라이언트는 `/api/image?p=<prompt>&s=<seed>&w=640&h=400` 만 호출
2. 서버 `/api/image/route.ts` 가 순차적으로 시도:
   - **1차**: Pollinations turbo 모델 (25초 타임아웃)
   - **2차**: Pollinations flux 모델 (35초, 품질 ↑)
   - **3차**: Picsum 실사진 폴백 (10초, always-works)
   - **최종 폴백**: 1x1 투명 PNG
3. 응답에 `cache-control: max-age=604800, immutable` 로 Next 캐시 활용

### 왜 프록시?
- 브라우저에서 직접 Pollinations 를 호출하면 CORS·긴 타임아웃·대용량 응답으로 실패가 잦았음
- 서버에서 받으면 Next 이미지 프록시가 일정 시간 대기 가능 + 캐시 가능 + 폴백 체인 실행 가능

### 프롬프트 규칙
- Claude 가 각 page 마다 영문 `imagePrompt` (30~60자) 생성
- 서버는 스타일 프리픽스 강제: `watercolor dreamy illustration, soft pastel palette, storybook style, muted colors, no text`
- 금지: 사람 얼굴 디테일·한국어 텍스트·실명·브랜드

### 결정적 이미지 (seed)
- 페이지마다 고정 `seed` 를 넣어 동일 꿈에서는 매번 같은 이미지가 나오도록
- 프리뷰 퀄리티 재현성 확보

### 이미지는 그림일기(premium) pages 에만
- **일반 해석(basic) 에는 이미지 없음** (텍스트만)
- interpretationBlocks 에도 이미지 없음 (해석 톤이 시각 자극에 눌리지 않게)

## 14.46 크레딧 결제 플로우 (모달 2-step)

## 14.47 배경 고정 (CloudLayer / StarField)

- CloudLayer · StarField 둘 다 **`position: fixed`** 로 viewport 에 고정.
- 이유: 드림피드처럼 무한 스크롤되는 콘텐츠에서 `position: absolute` 기반이면 배경이 부모 높이를 따라 같이 늘어나서 구름이 커져 보임. fixed 로 고정하면 피드를 아무리 내려도 구름 높이 일정.
- `z-index: 0` (콘텐츠 `z-index: 10` 보다 뒤)
- `pointerEvents: 'none'` 으로 상호작용 차단

5. 원문 토글 (premium)
6. 댓글 섹션 (목록 + 입력)

## 14.6 피드 & UX 룰

### 드림피드
- 정렬: 최신순
- 페이지네이션 없음. **IntersectionObserver 기반 무한 스크롤**
- 페이지당 4건, sentinel 이 viewport 진입 시 다음 페이지 로드
- 하단 48px 여백 + 끝 메시지 "모든 꿈을 다 보셨어요"

### 드림피드 vs 내 일기 구분
- 드림피드: `shared === true` 인 내 꿈 + 가상 공개 사용자 꿈
- 내 일기: 내 모든 꿈 (공개 여부 무관)
- 본인이 쓴 꿈도 드림피드에선 **닉네임**으로 표기 (하드코딩 "나" 금지)

### 설정 페이지
마이페이지 모달 대신 독립 탭(`settings`)으로 구성. 구성:
- 프로필 카드 (아바타 · 이름 · 이메일)
- 통계 3분할 (기록한 꿈 · 공개한 꿈 · 보유 크레딧)
- 닉네임 수정 (12자 제한)
- 메뉴: 충전 히스토리 / 내 일기 관리 / 약관

## 16.5 UI 이모지·아이콘 정책

- **표정·장식 이모지 전면 금지**: 😊 🤔 😱 😢 💎 🍀 ✨ 📝 등 모두 UI에서 사용 안 함.

### 예외
- 사용자 입력 본문(dream, 댓글 text) — 사용자가 쓴 이모지는 그대로 노출.
- 파스텔 컬러 블록(lucky color 칩) — 이모지 아닌 실제 색상.

## 18.0 Basic vs Premium 포맷 (반드시 구분)

### Basic 해석 (기본 해석, 5 크레딧)
- **데이터**: `interpretation: string` 필드 하나
- **구조**: §18.1 의 canonical 4단 포맷 (오프닝 / 상징 해석 / 심리적 의미 / 오늘의 조언)
- **볼드**: 소제목에는 금지. 본문 안 핵심 키워드에만 `**...**`
- **분량**: 500~900자
- **이미지 없음**
- **Lucky·avoid 없음**

### Premium 그림일기 (15 크레딧)
- **데이터**: `pages` (5장 그림일기) + `interpretationBlocks` (5~7개 해석 블록) + `lucky` (행운 아이템)
- **`pages`**: 사용자 꿈을 5장으로 쪼갠 그림일기. **반말 친근체**, 각 페이지에 AI 생성 이미지.
- **`interpretationBlocks`**: 존댓말 해석 블록. 각 블록은 heading + body. **이미지 없음.**
- **`lucky`**: 가지고 다닐 아이템 / 색상 / 피해야 할 것 / 방향 / 숫자
- 절대 basic 의 단일 `interpretation` 필드에 모든 걸 몰아넣지 말 것

### 중요한 구분
| 영역 | basic | premium |
|------|-------|---------|
| 해석 텍스트 | `interpretation` 한 덩어리 (canonical 포맷) | `interpretationBlocks` 배열 (6~7개) |
| 이미지 | 없음 | `pages` 5장에만 있음 |
| 반말/존댓말 | 존댓말 (canonical) | pages=반말, interpretationBlocks=존댓말 |
| Lucky | 없음 | 있음 (별도 카드) |

## 18.1 Canonical Basic 해석 예시 (시험 꿈)

모든 basic 해석은 아래 포맷을 따른다:

```
시험 꿈은 전 세계 공통으로 가장 많이 꾸는 꿈 중 하나예요. 학교를 졸업한 지 오래된 사람도 반복해서 꾸는데, "평가받는 상황에 대한 보편 불안"이 원형(archetype)처럼 자리잡아 있기 때문이에요.

상징 해석
• **글자가 뭉개져서 안 보인다**는 건 "기준이 명확하지 않은 평가"를 받고 있다는 감각이에요.
• **시계 초침이 너무 빠름**은 현실의 마감·조급함이 과장되어 나타난 거예요.
• **친구들은 태연한데 나만 못 쓴다**는 장면은 "다들 잘 하는데 나만 뒤처졌다"는 비교 불안의 정직한 표현이에요.

심리적 의미
이 꿈은 현재 당신이 받고 있는 외부 평가보다, 스스로에게 요구하는 내면의 기준이 지나치게 높다는 신호예요. 눈물이 안 나왔다는 건 그 긴장을 밖으로 풀 통로가 부족하다는 뜻이에요.

오늘의 조언
오늘 하루, 꼭 지키지 않아도 되는 "내가 만든 규칙" 하나를 스스로에게 용서해 주세요. 완벽함보다 "꾸준함"이 필요한 시기예요.
```

이 예시는 프롬프트의 few-shot 에 항상 포함. 길이·톤·소제목 스타일 모두 이 기준으로 맞춤.
